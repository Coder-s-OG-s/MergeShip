# Security Policy

## Supported Versions

Currently, the `main` branch of MergeShip is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

We take the security of MergeShip seriously. If you discover a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via a GitHub Private Vulnerability Report:

1. Go to the [Security tab](https://github.com/Coder-s-OG-s/MergeShip/security) of this repository.
2. Click on **Advisories** in the sidebar.
3. Click on **Report a vulnerability**.
4. Provide a detailed description of the vulnerability, including steps to reproduce it.

### Expected Response Timeline

The maintainer team will review your report and aim to respond to you within **48 hours**. We will keep you informed of our progress as we investigate and develop a fix.

### Responsible Disclosure Policy

- Please allow us a reasonable amount of time to resolve the issue before disclosing it to the public or a third party.
- We ask that you do not share details of the vulnerability publicly until a fix is available and has been deployed.
- We will notify you when the issue has been patched and credit you in the security advisory if you wish.
